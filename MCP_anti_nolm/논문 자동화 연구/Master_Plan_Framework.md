# 논문 자동화 연구 프레임워크 (Master Plan)

이 문서는 '클레오파트라 피부 관리법' 과제 수행 과정에서 검증된 **"AI 웹 리서치 + DBpia 학술 논문 하이브리드 전략"**을 기반으로, 앞으로 주어질 어떠한 주제의 서술형 과제라도 동일하게 고품질의 참고문헌을 확보하고 로직을 자동화할 수 있도록 설계된 범용 파이프라인 계획서입니다.

---

## 1. 프레임워크 설계 사상
순수 완전 자동화 프로세스는 유료 학술 DB(DBpia, RISS 등)의 보안/구독 장벽과 PDF 원본의 복잡성으로 인해 잦은 오류를 유발했습니다. 따라서 본 프레임워크는 **AI가 할 수 있는 지능적 데이터 파인딩 및 윤문**과 **사용자(인간)만이 뚫을 수 있는 인증 장벽**을 철저히 분업하는 'Semi-Automation(반자동)' 하이브리드 체제를 표준으로 삼습니다.

## 2. 이동식(Portable) 폴더 및 파일 구조 체계
새로운 주제의 과제가 주어질 때마다 `논문 자동화 연구` 폴더 하위에 아래와 같은 표준 트리 구조를 복제하여 사용합니다. 특히 이 폴더는 **다른 컴퓨터(PC)로 통째로 복사해서 옮겨도 자체적으로 파이썬 스크립트가 실행될 수 있도록 (Portable)** 의존성 설치 파일 및 실행 스크립트가 내장되어 있습니다.

```text
📁 논문 자동화 연구/
 ├── 📁 [주제01]_클레오파트라_미용/ (완료 레퍼런스)
 ├── 📁 [주제02]_새로운_과제_키워드/ (신규 생성 구조)
 │    ├── 📄 01_초안_원본.md                  (사용자가 작성한 최초 텍스트)
 │    ├── 📄 02_키워드_추출결과.md            (초안 분석 후 도출된 한/영 검색어 3~5세트)
 │    ├── 📄 03_타겟_논문_리스트.md           (DBpia API 검색 표 및 다운로드 가이드)
 │    ├── 📄 04_인용_양식.txt                 (요구 참고문헌 포맷팅)
 │    └── 📄 05_최종_융합본_하이브리드.md     (NotebookLM 윤문을 거친 최종 산출물)
 ├── 📁 Agent_Prompts/ (AI 서브 에이전트 지시서)
 │    └── � 13_에이전트프롬프트_참고문헌조사자.md (Step 1~3 원스톱 자동화 담당)
 └── �📁 Scripts_Lib/ (공통 스크립트 라이브러리 - 이동식 모듈)
      ├── 📄 dbpia_search.py                  (범용 DBpia API 검색기 파이썬 소스)
      ├── 📄 .env                             (API 인증 키 보관용)
      ├── 📄 requirements.txt                 (다른 PC 접속 시 필요한 파이썬 라이브러리 목록)
      └── 📄 setup_and_run.bat                (다른 PC에서 더블클릭 시 환경 세팅 및 실행해주는 배치 파일)
```

## 3. 표준 실행 파이프라인 (The 7-Step Workflow)

어떤 주제의 과제이든 무조건 아래 7단계의 플로우를 타고 진행됩니다. 특히 가장 손이 많이 가는 **Step 1~3은 `13_에이전트프롬프트_참고문헌조사자.md`를 탑재한 서브 에이전트에게 지시를 내리는 것만으로 한 번에 원스톱(One-Stop) 처리**됩니다.

| 단계 | 책임 주체 | 수행 내용 |
| :---: | :---: | :--- |
| **Step 1~3<br>(서브 에이전트 원스톱 리서치)** | **AI 서브 에이전트** | 사용자가 `Agent_Prompts/13_...md` 지시서를 부여하고 과제 주제를 던져주면, AI가 알아서 **① 초안 키워드 분석, ② 딥 리서치 자동 실행 및 노트북 임포트, ③ 파이썬 기반 DBpia 학술 검색 및 타겟 표 생성**까지 한 번에 쭉 실행하고 그 결과물만 사용자에게 표출합니다. |
| **Step 4<br>(다운로드)** | **사용자 (Human)** | (자동화 불가 구간) 사용자는 제공된 `03_타겟_논문_리스트.md`를 보고 맘에 드는 논문 1~2편의 링크를 눌러 기관 로그인 후 PDF를 직접 다운로드합니다. |
| **Step 5<br>(데이터 주입)** | **사용자 (Human)** | 사용자는 다운로드한 PDF 파일을 현재 작업 중인 NotebookLM 노트북 소스창에 드래그&드롭으로 수동 업로드하고, AI에게 "업로드 완료" 사인을 보냅니다. |
| **Step 6<br>(최종 융합)** | **AI + NLM** | AI는 프롬프트를 통해 NotebookLM에게 `01_초안_원본.md`와 새롭게 취합된 [웹 소스 + PDF 논문]의 데이터를 학술적으로 융합 윤문할 것을 지시합니다.<br>동시에 `04_인용_양식.txt`에 맞춰 하단 참고문헌 리스트를 교체, `05_최종_융합본_하이브리드.md`를 도출합니다. |
| **Step 7<br>(인용 검증)** | **AI + NLM** | 작성된 최종본의 핵심 문구들이 실제 업로드된 논문의 내용과 일치하는지 NotebookLM 채팅을 통해 팩트체크를 수행합니다. 그 결과를 `06_논문인용_검증리포트.md`로 기록하여 프로젝트를 최종 마감합니다. |

## 4. 다른 PC (이동형) 세팅 가이드
본 폴더를 USB나 구글 드라이브 등을 통해 동료의 PC나 다른 장소의 노트북으로 옮겼을 경우, 다음 1회성 절차만 수행하면 동일한 자동화 스크립트를 즉시 사용할 수 있습니다.
1. **사전 준비:** 새 컴퓨터에 Python 3.9 이상이 설치되어 있어야 합니다.
2. **원클릭 세팅:** `Scripts_Lib` 폴더 안에 있는 `setup_and_run.bat` 파일을 더블클릭합니다. (이 배치파일은 내부적으로 `pip install -r requirements.txt`를 자동 수행하여 aiohttp, requests 등의 의존성 통신 모듈을 새 PC에 자동으로 1초 만에 깔아줍니다.)
3. 기존처럼 AI 에이전트(혹은 Cursor/Terminal)를 통해 DBpia 검색 스크립트 명령(Step 3)을 마찰 없이 호출하면 됩니다.

## 5. 프레임워크의 확장성 (Future Work)
*   **다국어 문헌 확장성:** 현재 DBpia(한국어 타겟) 한정이지만, 향후 `Scripts_Lib` 폴더 내에 arXiv API(컴퓨터과학/수학), PubMed API(의학/바이오) 파이썬 스크립트를 추가하면 이공계 및 의약계열 과제까지 포괄하는 글로벌 논문 자동화 라우팅이 가능해집니다.
*   **서브 에이전트 특화:** 이전 실험에서 생성했던 `13_에이전트프롬프트_참고문헌조사자.md`를 이 폴더의 기본 프롬프트로 세팅하면, 매번 절차를 지시할 필요 없이 "키워드 추출하고 딥 리서치 및 논문 표 생성해" 라는 단 한 줄의 명령어로 Step 1~3이 완전 자동 연결 수행됩니다.
