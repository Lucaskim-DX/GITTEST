# DBpia 참고문헌 검색 프로그램 - 향후 개선 사항 (Future Improvements)

현재 작성된 `dbpia_search.py` 스크립트는 범용적인 검색, 페이징, 정렬, 발행연도 필터 기능을 제공하며 정상 작동합니다. 향후 프로그램의 완성도와 효율성을 높이기 위해 다음과 같은 추가 개선 사항을 적용해 볼 수 있습니다.

## 1. 성능 및 속도 향상 (비동기 처리 도입)
- **현상:** 여러 개의 키워드를 담은 텍스트 파일을 순차적(`for` 반복문)으로 하나씩 검색하고 있어 대량의 키워드 검색 시 시간이 소요됩니다.
- **개선 방안:** `asyncio` 및 `aiohttp` 라이브러리를 활용하여 여러 API 요청을 비동기(병렬)로 한 번에 처리하도록 코드를 리팩토링할 수 있습니다. 이를 통해 검색 속도가 기하급수적으로 빨라집니다.

## 2. API 예외 처리 고도화
- **현상:** `requests.raise_for_status()`를 통해 기본적인 HTTP 에러(400, 500 등)는 잡아내지만, DBpia API 자체의 상태 코드나 일시적 트래픽 초과(Rate Limit - HTTP 429) 시의 구체적 조치가 부족합니다.
- **개선 방안:** 트래픽 제한으로 튕겼을 때 잠시 대기했다가 다시 재시도하는 로직(Retry mechanism, Backoff 알고리즘)을 내장하여 검색의 안정성을 극대화합니다.

## 3. 결과 데이터 포맷 다양화 (JSON, Markdown)
- **현상:** 검색 결과가 CSV(`dbpia_results.csv`) 하나로만 고정되어 출력됩니다.
- **개선 방안:** 스크립트 실행 시 `--format` 옵션을 두어 `csv`, `json`, 혹은 과제 보고서에 바로 복사/붙여넣기 할 수 있는 `markdown` 표 형식 중 사용자가 원하는 형태로 자동 추출할 수 있게 기능을 추가합니다.

## 4. 정교한 필터링 옵션 확장
- **현상:** 단일 검색어(`searchall`), 정렬, 연도, 페이징 등 기본적인 필터만 구현되어 있습니다.
- **개선 방안:** DBpia 오픈 API 명세에서 지원하는 "발행기관명(`searchpublisher`)", "저자명(`searchauthor`)", "무료 제공 논문만 모아보기(`freeyn=Y`)" 등의 상세 선택 파라미터 옵션을 `argparse` 플래그로 추가 노출하여 학술적 타겟팅 검색을 돕습니다.

## 5. 중복 결과 제거 (Deduplication)
- **현상:** 다중 키워드 검색 시 서로 다른 키워드에 대해 동일한 논문이 검색되면 CSV 결과 쪽에 그대로 2번 쌓이게 됩니다.
- **개선 방안:** 결과를 병합하기 전에 논문의 `link_url`이나 `title`을 `Set` 구조로 검사하여 중복 수집된 논문은 1회만 기록되도록 내부 로직을 보완합니다.
